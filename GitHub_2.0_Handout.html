GitHub 2.0: The Handoff

The Upskilling Labs • Both partners do everything
YOU build → push to GitHub → PARTNER clones, branches, edits → opens PR → YOU review & merge → pull back
PARTNER builds → push to GitHub → YOU clone, branch, edit → open PR → PARTNER reviews & merge → pull back
Two full loops running in parallel. The page is bait — the process is the deliverable.

Build

2
Push
Git pane → Stage all → Commit → Push. Repo name: <yourname>-handoff-lab — PUBLIC.

Confirm

4
Prompt Doc

5
Invite
Settings → Collaborators → add your partner's GitHub username.
Prompt 1 — Fill in Your Blanks (full version in Google Drive)

Build a single-page personal website. Keep it SIMPLE: one page, no routing, no backend, no forms.

The page is for: ___________  |  Headline: "___________"
One sentence about me: "___________"
Three Things About Me cards: ___________, ___________, ___________

Keep all text easy to find in the code. No extra pages, animations, or heavy libraries.
✓ Both partners: repo live on GitHub, prompt doc committed, collaborator invite sent AND accepted.

~20 min
1
Accept
Accept your partner's collaborator invite (check email / GitHub notifications).
2
Clone
GitHub Desktop: File → Clone → find your partner's repo.
3
Branch
Current Branch → New Branch → feature/<yourname>-edit. Confirm you're NOT on main.
4
Read
Open prompts/scaffold-prompt.md — your map to the code.
🔍 Ctrl+Shift+F / Cmd+Shift+F Project-wide search — finds any text across all files. Your most important VS Code skill for unfamiliar code.

The Change Menu — Pick ONE

⭐ Text Swap

⭐⭐ Add a Fourth Card

⭐⭐⭐ AI Color Change
Find color hex values. Ask your AI assistant how to shift the palette. Save the exchange as prompts/edit-1.md.

Edit

Push

8

✓ Both partners: Pull Request open on your partner's repo. You should see the yellow banner on GitHub.com.

3

Review, Merge, and Sync Back

~12 min
1
Open PR
Go to YOUR repo on GitHub.com. You have a PR from your partner.
2
Review
Click "Files changed." Green = added. Red = removed. This is the diff.
3
Comment
Click a line number to comment on a specific line. Even "Looks good!" counts.
4
Merge
"Merge pull request" → Confirm → "Delete branch."
5
Sync
In Replit: Git pane → Pull. Your partner's change appears in your project.
✓ Loop complete. Code round-tripped: Builder → GitHub → Partner → Branch → PR → Main → Builder. Nothing broke.

The Handoff Protocol

Your build-cycle working agreement — keep this
1 Main always works. No direct commits to main. Ever.
2 Branch first, then edit. Check the branch indicator before you type.
3 One branch = one person = one task. Descriptive names: feature/..., fix/...
4 Every AI generation gets a prompt doc in /prompts, committed with the code.
5 Commits say who/how. Hand-edited vs. AI-generated, and which prompt doc to read.
6 All changes enter main through a Pull Request. At least one teammate reviews before merging.
7 Pull before you start; push before you stop.
8 The builder tool is personal; the repo is shared. GitHub is where the team meets.

When Things Go Wrong

"I can't push"
Click Pull first (someone else pushed before you). Then push again.
Editing on main by accident
In GitHub Desktop: Branch menu → New Branch. Your uncommitted changes move to the new branch.
Can't find the text in the code
Use Ctrl/Cmd+Shift+F (project-wide search). Check src/pages/ or src/components/ first.
Collaborator invite not arriving
Check email spam folder. Or go to github.com/notifications directly.
Replit sync looks stale
Open Replit Git pane → Pull. If still stale, make a small commit on GitHub.com to nudge it.
Out of Replit credits
Ask a mentor for a backup repo. You can skip straight to Round 2 — nobody sits out.

Essential Shortcuts

Ctrl/Cmd + Shift + F — Search all files (VS Code)
Ctrl/Cmd + S — Save file
Ctrl/Cmd + Z — Undo last edit
. on any GitHub repo — Open github.dev editor (Plan B!)
💡 Plan B: github.dev
If your local tools break, press . on any GitHub repo page. Full VS Code opens in the browser — edit, commit, push. No install needed.

Before Your First Build-Cycle Session

  1. One teammate creates the team repo and invites everyone.
  2. Everyone clones and does a test handoff — one commit on a branch with a PR.
  3. Write your scaffold prompt together. Commit to /prompts before anyone builds.
  4. Read the Handoff Protocol as a team. Out loud. These are your house rules.
